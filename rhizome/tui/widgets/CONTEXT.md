# rhizome/tui/widgets/

Reusable Textual widgets shared across screens.

## Files

- **chat_pane.py** — `ChatPane(Widget)`: core chat UI containing the message area, chat input, and command palette. Holds per-session state (`messages`, `_agent_busy`, `_agent_worker`, `session_mode`, `active_topic`, `_agent_session`) and all message/command/topic-tree handlers. Each `ChatPane` instance represents an independent chat session with its own `AgentSession` and `CommandRegistry`; multiple instances live inside `TabbedContent` tabs in `ChatScreen`. Delegates to `AgentSession.stream()` (callback-driven) for agent invocation, passing `AgentMessageHarness` callback methods directly. Handles `InterruptPending`/`InterruptResolved` messages from the harness to disable/re-enable the chat input during interrupt waits. Mirrors user/system messages into the session via `append_message()`. Command handlers live as `_cmd_*` methods (e.g. `_cmd_options`, `_cmd_rename`, `_cmd_clear`); `_set_mode()` handles mode switching and is called by both slash commands and agent tools. `_register_commands()` wires all handlers into the click-based `CommandRegistry` with proper option/argument schemas. `_handle_command()` dispatches through the registry, showing help/error text when returned.
- **status_bar.py** — `StatusBar(Static)`: persistent bar at the bottom of the chat screen with reactive `mode`, `context`, and `token_usage` (a `TokenUsageData` instance) properties. The token display splits conversation tokens from overhead (system prompt), shown as `tokens: 1,234 (+328)  context usage: 2.3%` where `(+N)` is rendered in a lighter gray.
- **message.py** — Chat message widget hierarchy. `ChatMessage(Widget)` is the abstract base class handling role prefixes, CSS styling, and collapse logic. Two concrete subclasses: `MarkdownChatMessage` renders content via Textual's `Markdown` widget (used for user/agent/system messages), and `RichChatMessage` renders content via `Static(Text.from_ansi(...))` for proper display of ANSI escape codes (used for command output like help text). User messages have a dark gray background with side margins; agent messages gain a thin rounded border in learn/review mode. Agent messages longer than `COLLAPSE_LINE_THRESHOLD` (4) lines show a collapse/expand button.
- **thinking.py** — `ThinkingIndicator(Static)`: animated braille-spinner shown while awaiting the agent's first token.
- **topic_tree.py** — `TopicTree(Tree[Topic])`: interactive topic browser. Loads root topics on mount, lazily loads children on expand. Posts `TopicSelected(topic)` on Enter and `Dismissed()` on Ctrl+Enter.
- **agent_message_harness.py** — `AgentMessageHarness(Widget)`: encapsulates one agent turn's display lifecycle. Supports arbitrary interleaving of `ChatMessage`, `ToolCallList`, and `InterruptChoices` segments via `_segments`. A new segment is created at each text↔tool boundary: `append(token)` starts a new `ChatMessage` if the last segment is a `ToolCallList` (or empty), and `post_update(chunk)` starts a new `ToolCallList` if the last segment is a `ChatMessage` (or empty), closing the active `MarkdownStream` in the process. Exposes callback methods (`on_message`, `on_update`, `on_interrupt`) matching the signatures expected by `AgentSession.stream()`. `on_interrupt` mounts an `InterruptChoices` widget and blocks (via `asyncio.Future`) until the user responds; it posts `InterruptPending`/`InterruptResolved` Textual messages so `ChatPane` can toggle input. Methods: `start_thinking()`, `stop_thinking()`, `append(token)`, `post_update(chunk)`, `finalize() -> str`, `cancel() -> str`. Properties: `chat_message_body` (joined text from all `ChatMessage` segments), `is_thinking`. On finalization, calls `update_body()` on each `ChatMessage` segment to trigger collapsibility checks. Mounted synchronously by `ChatPane._handle_chat()` before the agent worker starts; the worker then drives the harness through its lifecycle. No background or border styling of its own — mode-specific borders are applied at the individual `ChatMessage` level.
- **interrupt_choices.py** — `InterruptChoices(Widget)`: displays an interrupt prompt with option buttons, backed by an `asyncio.Future`. Mounted by `AgentMessageHarness.on_interrupt()`. `wait_for_selection()` awaits the future; button presses resolve it. `cancel()` cancels the future for Ctrl+C cleanup.
- **tool_call_list.py** — `ToolCallList(Widget)`: collapsible list of tool call names with Unicode box-drawing characters (`├──` / `└──`). No background or border — uses a `Static` title widget ("tool calls ▼") instead of `border_title`, with left padding aligned to agent message text. Mounted lazily by `AgentMessageHarness.post_update()` when the agent invokes tools. `add_tool(name)` appends a name and re-renders the full list. Click or Enter toggles collapse: expanded shows the tree; collapsed hides content and shows "N tool calls (click to expand...) ▶".
- **chat_pane.py (commit mode)** — Commit selection is handled inline by `ChatPane` via a `_commit_mode` flag rather than a separate widget. `enter_commit_mode()` scans for `ChatMessage.agent-message.learn-mode` widgets, decorates them with checkboxes and CSS classes (`--commit-selectable`, `--commit-cursor`, `--commit-selected`), and disables the chat input. `on_key()` intercepts Up/Down/Space/Ctrl+J/Escape during commit mode. `exit_commit_mode()` cleans up decorations and restores normal input. Invoked by the `/commit` command.
- **logging_pane.py** — `LoggingPane(Widget)`: displays a rolling window of log messages captured by `TUILogHandler`. Composes a single `RichLog(max_lines=2000, markup=True, wrap=True, auto_scroll=True)`. On mount, replays all existing lines from the handler's deque and registers itself for live updates. Used inside `LogTabPane` in the chat screen, opened via the `/logs` command.
- **command_palette.py** — `CommandPalette(Widget)`: autocomplete dropdown for slash commands. Dynamically sources its command list from the parent `ChatPane`'s `CommandRegistry` (walking the widget tree to find it). Shown when input starts with `/`, filters by prefix, supports arrow-key navigation and Tab/Enter selection. Posts `CommandSelected(name)` when a command is chosen.
