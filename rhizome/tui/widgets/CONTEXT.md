# rhizome/tui/widgets/

Reusable Textual widgets shared across screens.

## Files

- **chat_pane.py** — `ChatPane(Widget)`: core chat UI containing the message area, chat input, and command palette. Holds per-session state (`messages`, `_agent_busy`, `_agent_worker`, `session_mode`, `active_topic`, `_agent_session`) and all message/command/topic-tree handlers. Each `ChatPane` instance represents an independent chat session with its own `AgentSession` and `CommandRegistry`; multiple instances live inside `TabbedContent` tabs in `MainScreen`. Delegates to `AgentSession.stream()` (callback-driven) for agent invocation, passing `AgentMessageHarness` callback methods directly. Handles `InterruptPending`/`InterruptResolved` messages from the harness to disable/re-enable the chat input during interrupt waits. Mirrors user/system messages into the session via `append_message()`. Command handlers live as `_cmd_*` methods (e.g. `_cmd_options`, `_cmd_rename`, `_cmd_clear`); `_set_mode()` handles mode switching and is called by both slash commands and agent tools. `_register_commands()` wires all handlers into the click-based `CommandRegistry` with proper option/argument schemas. `_handle_command()` dispatches through the registry, showing help/error text when returned.
- **status_bar.py** — `StatusBar(Static)`: persistent bar at the bottom of the chat screen with reactive `mode`, `topic_path`, `token_usage`, and `model_name` properties. Three-line layout: line 1 shows topic path (left) and model name (right), line 2 shows mode with shift+tab hint (left) and token usage with overhead breakdown (right), line 3 shows cache read/create stats (right-aligned).
- **message.py** — Chat message widget hierarchy. `ChatMessage(Widget)` is the abstract base class handling role prefixes, CSS styling, and collapse logic. Two concrete subclasses: `MarkdownChatMessage` renders content via Textual's `Markdown` widget (used for user/agent/system messages), and `RichChatMessage` renders content via `Static(Text.from_ansi(...))` for proper display of ANSI escape codes (used for command output like help text). User messages have a dark gray background with side margins; agent messages gain a thin rounded border in learn/review mode. Agent messages longer than `COLLAPSE_LINE_THRESHOLD` (4) lines show a collapse/expand button.
- **thinking.py** — `ThinkingIndicator(Static)`: animated braille-spinner shown while awaiting the agent's first token.
- **entry_list.py** — `EntryList(Widget)`: read-only, focusable widget for browsing a list of `KnowledgeEntry` objects. Displays a numbered entry list with cursor navigation and a bordered detail panel (title, meta, content). `set_entries()` replaces the displayed entries. Supports a `--compact` CSS class that hides the detail panel and numbering, rendering minimal `► title` lines with a metadata hint below (type, created date). Posts `Dismissed()` on Escape and `CursorChanged(entry)` when the cursor moves. Also exports shared color constants (`ENTRY_DIM`, `ENTRY_HINT`, `ENTRY_ACCENT`) used by both this widget and `CommitProposal`.
- **flashcard_list.py** — `FlashcardList(Widget)`: read-only, focusable widget for browsing a list of `Flashcard` objects. Similar structure to `EntryList`: numbered list with cursor navigation and a detail panel showing question, answer, testing notes, and linked entry count. Ephemeral flashcards display an `(ephemeral)` suffix in dim color. `set_flashcards()` replaces the displayed flashcards. Posts `Dismissed()` on Escape.
- **flashcard_viewer.py** — `FlashcardViewer(Widget)`: interactive flashcard viewer widget with reveal-and-rate flow. Accepts a list of `FlashcardItem` dicts (question/answer). Displays a centered card with question on top; pressing Space reveals the answer and rating options (0-again, 1-hard, 2-good, 3-easy). Space again counts as "good". Rating 0 reinserts the card at the end of the queue. Ctrl+Left/Right navigates between cards; header shows navigation hint (left) and index/total (right). Posts `Dismissed()` on Escape, `CardRated(question, rating, rating_label)` on each rating, and `SessionComplete()` when the queue is empty. Mounted by the `/test-flashcards` command with sample data.
- **topic_tree.py** — `TopicTree(Tree[Topic])`: the core interactive tree widget for browsing topics. Loads root topics on mount, lazily loads children on expand. Enter is suppressed (topic selection is handled by Ctrl+J/Ctrl+Enter in the parent container). `TopicTreeViewer(Vertical)`: legacy bordered container (superseded by `ExplorerViewer`).
- **explorer_viewer.py** — `ExplorerViewer(Vertical)`: bordered container for browsing topics, entries, and flashcards. Composes `TopicTree`, `EntryList`, and `FlashcardList` inside labeled panes (titles shown in multi-pane modes). Four view modes cycled by Tab: topics only, topics & entries (30/70), topics & flashcards (30/70), topics + entries + flashcards (20/35/45). Ctrl+Left/Right navigates between visible panes; Ctrl+Enter selects a topic. In 3-pane mode, EntryList uses compact mode and selecting an entry loads its flashcards in the right panel. Cursor persistence: entry cursor shared across all entry-showing modes per topic; flashcard cursor persisted per topic in two-pane mode only; three-pane flashcard cursor resets on each entry selection. Posts `TopicSelected(topic, path)` and `Dismissed()`. Mounted by the `/explore` command.
- **agent_message_harness.py** — `AgentMessageHarness(Widget)`: encapsulates one agent turn's display lifecycle. Supports arbitrary interleaving of `ChatMessage`, `ToolCallList`, and `InterruptChoices` segments via `_segments`. A new segment is created at each text↔tool boundary: `append(token)` starts a new `ChatMessage` if the last segment is a `ToolCallList` (or empty), and `post_update(chunk)` starts a new `ToolCallList` if the last segment is a `ChatMessage` (or empty), closing the active `MarkdownStream` in the process. Exposes callback methods (`on_message`, `on_update`, `on_interrupt`) matching the signatures expected by `AgentSession.stream()`. `on_interrupt` mounts an `InterruptChoices` widget and blocks (via `asyncio.Future`) until the user responds; it posts `InterruptPending`/`InterruptResolved` Textual messages so `ChatPane` can toggle input. Methods: `start_thinking()`, `stop_thinking()`, `append(token)`, `post_update(chunk)`, `finalize() -> str`, `cancel() -> str`. Properties: `chat_message_body` (joined text from all `ChatMessage` segments), `is_thinking`. On finalization, calls `update_body()` on each `ChatMessage` segment to trigger collapsibility checks. Mounted synchronously by `ChatPane._start_agent()` before the agent worker starts; the worker (`_run_agent`) then drives the harness through its lifecycle. No background or border styling of its own — mode-specific borders are applied at the individual `ChatMessage` level.
- **interrupt.py** — `InterruptWidget` protocol defining the interface all interrupt widgets must satisfy: `wait_for_selection() -> Any` and `cancel()`.
- **choices.py** — `Choices(Widget)`: displays an interrupt prompt with a navigable list of options, backed by an `asyncio.Future`. Mounted by `AgentMessageHarness.on_interrupt()` for `"choices"` type interrupts. `wait_for_selection()` awaits the future; selection resolves it. `cancel()` cancels the future for Ctrl+C cleanup.
- **multiple_choices.py** — `MultipleChoices(Widget)`: multi-question interrupt widget with tabbed navigation. Presents multiple questions as a horizontal tab bar with checkbox indicators (`[x]`/`[ ]`). Ctrl+Left/Right navigates between questions; Up/Down/Enter selects options within each question. Answering a question marks its checkbox and auto-advances to the next unanswered one. When all questions are answered, Ctrl+Enter shows a "Submit answers?" confirmation (Yes/No). Resolves the future with a `dict[str, str]` mapping question names to selected options. Mounted for `"multiple_choice"` type interrupts.
- **sql_confirmation.py** — `SqlConfirmation(Widget)`: interrupt widget for confirming SQL modification statements. Displays a warning header with row count, the SQL statement in a styled block, a preview table of affected rows (for UPDATE/DELETE), and Approve/Deny options. Follows the same pattern as `WarningChoices`: asyncio.Future for blocking, `from_interrupt` classmethod, Up/Down/Enter navigation, collapse on selection. Mounted for `"sql_confirmation"` type interrupts.
- **warning.py** — `WarningChoices(Widget)`: warning interrupt for irreversible agent actions. Displays a warning icon (⚠) and highlighted message in amber, with Approve/Deny as default options plus any additional options from the interrupt config. After selection, the warning text is hidden and only the chosen option remains visible. Mounted for `"warning"` type interrupts.
- **commit_proposal.py** — `CommitProposal(Widget)`: specialized interrupt widget for reviewing commit proposals. Master-detail layout with an entry list (navigable), detail panel (editable title/content), and choice list (Approve All/Edit/Cancel). Supports inline editing (Enter to edit, Escape to return), entry exclusion toggle (`d`), type cycling (`f`), and topic selection via modal (`t`). Mounted for `"commit_proposal"` type interrupts.
- **tool_call_list.py** — `ToolCallList(Widget)`: collapsible list of tool call names with Unicode box-drawing characters (`├──` / `└──`). No background or border — uses a `Static` title widget ("tool calls ▼") instead of `border_title`, with left padding aligned to agent message text. Mounted lazily by `AgentMessageHarness.post_update()` when the agent invokes tools. `add_tool(name)` appends a name and re-renders the full list. Click or Enter toggles collapse: expanded shows the tree; collapsed hides content and shows "N tool calls (click to expand...) ▶".
- **chat_pane.py (commit mode)** — Commit selection is handled inline by `ChatPane` via a `_commit_mode` flag rather than a separate widget. `enter_commit_mode()` scans for `ChatMessage.agent-message.learn-mode` widgets, decorates them with checkboxes and CSS classes (`--commit-selectable`, `--commit-cursor`, `--commit-selected`), and disables the chat input. `on_key()` intercepts Up/Down/Space/Ctrl+J/Escape during commit mode. `exit_commit_mode()` cleans up decorations and restores normal input. Invoked by the `/commit` command.
- **logging_pane.py** — `LoggingPane(Widget)`: displays a rolling window of log messages captured by `TUILogHandler`. Composes a `RichLog(max_lines=2000, markup=True, wrap=True, auto_scroll=True)` and a `LogsStatusBar` hint bar docked at the bottom showing "ctrl+g to open in editor". On mount, replays all existing lines from the handler's deque and registers itself for live updates. Used inside `LogTabPane` in the chat screen, opened via the `/logs` command.
- **command_palette.py** — `CommandPalette(Widget)`: autocomplete dropdown for slash commands. Dynamically sources its command list from the parent `ChatPane`'s `CommandRegistry` (walking the widget tree to find it). Shown when input starts with `/`, filters by prefix, supports arrow-key navigation and Tab/Enter selection. Posts `CommandSelected(name)` when a command is chosen.
