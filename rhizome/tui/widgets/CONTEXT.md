# rhizome/tui/widgets/

Reusable Textual widgets shared across screens.

## Files

- **chat_pane.py** — `ChatPane(Widget)`: core chat UI containing the message area, chat input, and command palette. Holds per-session state (`messages`, `_agent_busy`, `_agent_worker`, `session_mode`, `session_context`, `active_curriculum`, `active_topic`) and all message/command/topic-tree handlers. Each `ChatPane` instance represents an independent chat session; multiple instances live inside `TabbedContent` tabs in `ChatScreen`.
- **status_bar.py** — `StatusBar(Static)`: persistent bar at the bottom of the chat screen with reactive `mode`, `context`, and `token_usage` (a `TokenUsageData` instance) properties. The token display splits conversation tokens from overhead (system prompt + app-generated `[System]` messages), shown as `tokens: 1,234 (+328)  context usage: 2.3%` where `(+N)` is rendered in a lighter gray. Overhead is computed after each agent turn via `build_lc_messages` + `count_tokens_approximately`.
- **message.py** — `ChatMessage(Widget)`: renders a single chat message with markdown support, a role prefix (`you` / `agent`), and role-based CSS class for styling. User messages have a dark gray background with side margins; agent messages have no background in idle mode but gain a thin rounded border in learn (blue) or review (purple) mode; system messages have no background. Agent messages longer than `COLLAPSE_LINE_THRESHOLD` (4) lines show a collapse/expand button; shorter messages hide it.
- **thinking.py** — `ThinkingIndicator(Static)`: animated braille-spinner shown while awaiting the agent's first token.
- **topic_tree.py** — `TopicTree(Tree[Topic])`: interactive topic browser. Loads root topics on mount, lazily loads children on expand. Posts `TopicSelected(topic)` on Enter and `Dismissed()` on Ctrl+Enter.
- **agent_message_harness.py** — `AgentMessageHarness(Widget)`: encapsulates one agent turn's display lifecycle. Supports arbitrary interleaving of `ChatMessage` and `ToolCallList` segments via `_segments: list[ChatMessage | ToolCallList]`. A new segment is created at each text↔tool boundary: `append(token)` starts a new `ChatMessage` if the last segment is a `ToolCallList` (or empty), and `post_update(chunk)` starts a new `ToolCallList` if the last segment is a `ChatMessage` (or empty), closing the active `MarkdownStream` in the process. Methods: `start_thinking()`, `stop_thinking()`, `append(token)`, `post_update(chunk)`, `finalize() -> str`, `cancel() -> str`. Properties: `chat_message_body` (joined text from all `ChatMessage` segments), `is_thinking`. On finalization, calls `update_body()` on each `ChatMessage` segment to trigger collapsibility checks. Mounted synchronously by `ChatPane._handle_chat()` before the agent worker starts; the worker then drives the harness through its lifecycle. No background or border styling of its own — mode-specific borders are applied at the individual `ChatMessage` level.
- **tool_call_list.py** — `ToolCallList(Widget)`: collapsible list of tool call names with Unicode box-drawing characters (`├──` / `└──`). No background or border — uses a `Static` title widget ("tool calls ▼") instead of `border_title`, with left padding aligned to agent message text. Mounted lazily by `AgentMessageHarness.post_update()` when the agent invokes tools. `add_tool(name)` appends a name and re-renders the full list. Click or Enter toggles collapse: expanded shows the tree; collapsed hides content and shows "N tool calls (click to expand...) ▶".
- **chat_pane.py (commit mode)** — Commit selection is handled inline by `ChatPane` via a `_commit_mode` flag rather than a separate widget. `enter_commit_mode()` scans for `ChatMessage.agent-message.learn-mode` widgets, decorates them with checkboxes and CSS classes (`--commit-selectable`, `--commit-cursor`, `--commit-selected`), and disables the chat input. `on_key()` intercepts Up/Down/Space/Ctrl+J/Escape during commit mode. `exit_commit_mode()` cleans up decorations and restores normal input. Invoked by the `/commit` command.
- **command_palette.py** — `CommandPalette(Widget)`: autocomplete dropdown for slash commands. Shown when input starts with `/`, filters by prefix, supports arrow-key navigation and Tab/Enter selection. Posts `CommandSelected(name)` when a command is chosen.
